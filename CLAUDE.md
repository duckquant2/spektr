# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Что это

«СПЕКТР» — научно-популярный журнал об аутизме. Каждый выпуск — **один markdown-файл**
`vypuski/spektr_vypusk_NN.md`, из которого pandoc собирает самодостаточную веб-страницу
`html/spektr_vypusk_NN.html` (один файл, без внешних зависимостей: шрифты системные, стили и
JSON-LD встроены в шаблон). Исходники `.md` лежат в `vypuski/`, готовые страницы — в `html/`.
Контент ведёт нетехнический редактор; правится только `.md`.

## Сборка

```bash
./builder/build.sh                       # собирает spektr_vypusk_01.md по умолчанию
./builder/build.sh spektr_vypusk_02.md   # собирает указанный выпуск
```

Запускать из корня репозитория (где лежат `vypuski/` и `builder/`); исходник берётся из
`vypuski/`, готовый `.html` пишется в `html/`. Требуется pandoc 3.x в PATH. Тестов нет —
проверка визуальная: открыть `html/spektr_vypusk_NN.html` в браузере либо поднять
`python -m http.server 8765` (конфиг в `.claude/launch.json`, адрес
`http://localhost:8765/html/spektr_vypusk_01.html`).

YAML-шапка `.md` (`issue`, `date`, `date-human`, `pagetitle`, `description`, `deck`) — источник
всех метаданных; кириллица берётся из файла в UTF-8, а **не** из аргументов командной строки —
так специально сделано, иначе она ломается на Windows. Файлы `.md` обязаны быть в UTF-8.

## Архитектура сборки

Три файла в `builder/` работают вместе, превращая обычный markdown в фирменную семантику макета:

- **`build.sh`** — вызывает pandoc с шаблоном, lua-фильтром и `--section-divs`
  (каждая рубрика оборачивается в `<section id>`).
- **`spektr-web.html5`** — pandoc-шаблон: весь `<head>`, встроенный `<style>` (дизайн-токены,
  светлая/тёмная тема, печатные стили `@media print`), обложка-masthead и **жёстко зашитое
  оглавление из 8 рубрик**. Тело выпуска подставляется в `$body$`.
- **`spektr-filters.lua`** — преобразует markdown-конструкции в классы макета:
  - таблица → `<div class="table-scroll">` (горизонтальный скролл на узких экранах);
  - КАПС-строка с разделителем `·` сразу под заголовком → `<div class="kicker">`;
  - абзац ровно из одного `**жирного**` фрагмента → заголовок-лид `<h4>`;
  - цитата `>`, начинающаяся с жирного зачина (или ставшая `<h4>`) → `<div class="callout">`
    (врезка-пояснение); обычная цитата остаётся `<blockquote>`, а её хвостовая строка с тире
    `— Имя, роль` выносится в `<footer class="attribution">`.

### Критическая связка: оглавление ↔ якоря рубрик

Оглавление и JSON-LD в `spektr-web.html5` ссылаются на **фиксированные id**:
`#от-редакции`, `#главное-за-месяц`, `#наука-понятным-языком`, `#подходы-и-практика`,
`#живые-истории`, `#культура-и-медиа`, `#родителю-на-заметку`, `#источники`.
Заголовки рубрик в `.md` задают этот id явно: `## 01 · От редакции {#от-редакции}`.
Меняя названия рубрик или их состав, синхронизируй три места: `{#...}` в `.md`, список `<ol>`
в шаблоне и `itemListElement` в JSON-LD — иначе оглавление будет вести в никуда.

## Конвенции редактора (правила разметки)

Эти три привычки нужны, чтобы фильтр собрал фирменные элементы (детали и примеры — в `README.md`):

1. **Кикер** — строка-ярлык под `### заголовком`: первое слово КАПСОМ + разделитель `·`
   (`РОССИЯ · клинические рекомендации`). Без `·` фильтр не сработает.
2. **Врезка-пояснение** — цитата `>` с жирным зачином: `> **Что это значит.** …`.
   Для списка внутри — пустая строка `>` перед пунктами.
3. **Подпись к цитате** — последняя строка цитаты с тире: `> — Имя Фамилия, роль`.
   Имя до запятой → `<cite>`, остальное → роль.

Lua-фильтр разбирает кикеры и подписи по **структуре инлайнов**, а не байтовым поиском —
паттерны Lua не Unicode-безопасны и режут кириллицу. Сохраняй этот подход при правках фильтра.
