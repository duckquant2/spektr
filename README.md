# СПЕКТР — как собрать выпуск

Журнал собирается **из одного текстового файла** `spektr_vypusk_NN.md` в готовую
веб-страницу `spektr_vypusk_NN.html`: один самодостаточный файл, читается с
телефона и ПК, печатается дома через Ctrl/Cmd+P. Внешних зависимостей нет —
шрифты системные, стили и оформление встроены.

**Главное правило:** правим только `.md`. Всё оформление (радужная линейка,
цвета, обложка, оглавление, печать, тёмная тема) собирается автоматически —
руками HTML трогать не нужно.

---

## Что лежит в проекте

| Папка / файл | Назначение | Трогать? |
|---|---|---|
| `vypuski/` | Исходники выпусков — текст в markdown (`spektr_vypusk_NN.md`) | **да, тут пишем** |
| `html/` | Готовые страницы — собираются автоматически (`spektr_vypusk_NN.html`) | нет |
| `builder/build.sh` | Скрипт сборки: из `.md` делает `.html` | нет |
| `builder/spektr-web.html5` | Шаблон-оболочка: `<head>`, стили, обложка, оглавление | редко |
| `builder/spektr-filters.lua` | Автоматика: кикеры, врезки, подписи цитат, таблицы | нет |
| `README.md` | Эта инструкция | — |

Мы пишем текст в `vypuski/`, а сборка кладёт готовый журнал в `html/`.

---

## Разовая настройка (один раз на компьютере)

1. **Установить Pandoc** — программу, которая собирает страницу.
   Открой **PowerShell** и выполни:

   ```powershell
   winget install JohnMacFarlane.Pandoc
   ```

   (или скачай установщик с <https://pandoc.org/installing.html> и пройди «Далее–Далее».)

2. **Закрой и открой терминал заново** — это важно, иначе система ещё не «видит»
   только что установленный Pandoc.

3. **Проверь**, что всё встало. В новом терминале:

   ```bash
   pandoc --version
   ```

   Если показалась строчка вида `pandoc 3.x` — готово.

Git Bash (терминал, в котором запускается сборка) обычно уже стоит вместе с Git.
Если его нет — поставь Git for Windows: <https://git-scm.com/download/win>.

---

## Как собрать выпуск

1. Отредактируй текст в файле `vypuski/spektr_vypusk_01.md` (см. правила ниже).
2. В папке `spektr` нажми правой кнопкой → **«Git Bash Here»**.
3. Введи команду и нажми Enter:

   ```bash
   ./builder/build.sh spektr_vypusk_01.md
   ```

   Для следующего выпуска — соответственно `spektr_vypusk_02.md` и т.д.
   (имя файла — без папки, скрипт сам берёт его из `vypuski/`).
4. Появится строка `Готово: html/spektr_vypusk_01.html`. Открой этот файл из
   папки `html/` двойным кликом — он откроется в браузере. Это и есть готовый журнал.

> Если набрать просто `./builder/build.sh` без имени файла — соберётся
> `spektr_vypusk_01.md` по умолчанию.

## Что менять в каждом новом выпуске

В самом верху `.md`-файла есть «шапка» между строками `---`. Это единственное
техническое место — поменяй в нём номер, дату и описание:

```yaml
---
pagetitle: "СПЕКТР · Научно-популярный журнал об аутизме · Выпуск №2"
description: "СПЕКТР — журнал об аутизме для родителей. Выпуск №2: ..."
issue: 2
date: "2026-07-22"
date-human: "22 июля 2026 года"
deck: "Научно-популярный журнал об аутизме · Выпуск №2"
lang: ru
---
```

- `issue` — номер выпуска (число).
- `date` — дата в формате `ГГГГ-ММ-ДД` (для техники).
- `date-human` — дата словами (её видит читатель).
- `pagetitle`, `description`, `deck` — заголовок вкладки, описание для поиска и
  подзаголовок на обложке.

Названия восьми рубрик (От редакции, Главное за месяц …) и оглавление —
постоянные, менять их не нужно.

## Три правила разметки (чтобы оформление собралось само)

Текст пишется обычным markdown. Чтобы получились фирменные элементы журнала,
держись трёх привычек — они уже видны в выпуске №1, можно просто копировать
по образцу.

**1. Кикер — строчка-ярлык под заголовком новости.**
Пиши первое слово КАПСОМ и ставь разделитель «·»:

```markdown
### 1. Заголовок новости

РОССИЯ · клинические рекомендации

Текст новости…
```

**2. Врезка-пояснение** (цветная карточка «на полях») — это цитата `>`,
начинающаяся с **жирного** зачина:

```markdown
> **Что это значит для практики.** Текст пояснения…
```

Если внутри врезки нужен список — оставь пустую строку с `>`:

```markdown
> **Приложения:**
>
> - **LetMeTalk** — бесплатное…
> - **PECS IV+** — до 750 слов…
```

**3. Подпись к цитате человека** — последняя строка цитаты, начинается с тире.
Имя до запятой станет именем автора, остальное — ролью/источником:

```markdown
> «Текст прямой речи человека.»
> — Имя Фамилия, роль (источник)
```

Обычная цитата (с «ёлочками» и без жирного зачина) останется цитатой, а
строка с тире превратится в аккуратную подпись.

## Печать дома

Открой готовый `.html` из папки `html/` в браузере → Ctrl/Cmd+P:

- **Бумага:** A4, поля «по умолчанию» (в стилях уже заданы 18×16 мм).
- **Фон:** галку «Графика фона / Background graphics» можно НЕ ставить —
  печатный режим убирает серые заливки врезок ради экономии тонера.
- **«Сохранить как PDF»** — если нужен PDF вместо принтера.
- Каждая рубрика (01–08) начинается с новой страницы; обложка — отдельная.

## Если что-то пошло не так

- **`pandoc: command not found`** — Pandoc не установлен или терминал открыт
  до установки. Закрой терминал, открой заново (см. «Разовая настройка»).
- **`build.sh: No such file or directory`** — терминал открыт не в той папке.
  Открывай Git Bash именно в корне `spektr` (где лежат папки `vypuski/` и `builder/`).
- **Заголовок вкладки в кракозябрах** — проверь, что `.md` сохранён в кодировке
  **UTF-8** (Блокнот: «Сохранить как» → Кодировка: UTF-8).
- **Кикер/врезка не оформились** — перечитай «Три правила разметки»: вероятно,
  пропал «·», жирный зачин или пустая строка с `>` перед списком.
